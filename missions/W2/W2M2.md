# 파이썬 멀티프로세싱 기초: Process 객체 직접 제어 실습

이번 과제는 파이썬의 `multiprocessing` 모듈 내 `Process` 클래스를 사용하여, 각각 다른 대륙 이름을 출력하는 독립적인 프로세스들을 직접 생성하고 동시에 실행해보는 실습 과제이다.

## 이번 프로젝트를 하며 배운 점들

**1. `multiprocessing.Process`를 통한 직접적인 프로세스 제어**
앞선 과제의 `Pool` 방식과 달리, 각 작업(Task)마다 1:1로 `Process` 객체를 직접 생성(Spawn)하는 저수준(Low-level) 제어 방식을 배웠다. 이를 통해 C언어의 스레드 프로그래밍(`pthread`)과 유사하게, 백그라운드 프로세스를 수동으로 하나하나 찍어내고 통제하는 감각을 익혔다.

**2. `.start()`와 `.join()`을 활용한 실행 및 흐름 제어**
생성된 프로세스 객체들을 리스트에 담아둔 뒤, 첫 번째 반복문에서 `.start()`를 호출해 모든 프로세스를 동시에 백그라운드로 띄우는 방법을 경험했다. 또한 두 번째 반복문에서 `.join()`을 호출함으로써, 메인 프로세스(부모)가 자식 프로세스들의 작업이 모두 끝날 때까지 종료되지 않고 안전하게 기다리게 만드는 동기화(Synchronization) 기법을 직접 구현해 보았다.

**3. 함수의 기본 인자(Default Argument)와 `args` 튜플 전달**
대륙 이름을 출력하는 단일 함수에 인자를 주지 않을 때는 기본값("Asia")이 출력되도록 설정하고, 나머지 작업들은 `args=(continent,)` 형태의 튜플로 각각 다른 데이터("America", "Europe", "Africa")를 넘겨주었다. 이를 통해 병렬 처리 시에도 단일 함수를 재사용하여 다양한 결괏값을 도출하는 방법을 확인했다.

**4. `Process` 방식과 `Pool` 방식의 차이점 확립 (구조적 이해)**
이번 과제를 통해 소수의 정해진 독립 작업(4개 프로세스)을 띄울 때는 `Process` 객체를 직접 생성하는 것이 직관적이라는 것을 배웠다. 나아가 이전 `Pool` 과제와 비교해보며, 작업량이 방대할 경우 프로세스 생성 오버헤드를 막기 위해 `Pool`을 써야 하고, 독립적이고 소수인 작업을 직접 통제할 때는 `Process`를 써야 한다는 **'상황에 따른 설계의 차이'**를 명확하게 이해하게 되었다.
