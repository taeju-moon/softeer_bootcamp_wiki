# Jupyter Notebook Docker 배포 실습: W1M1, W1M2, W2M5 컨테이너화

이번 과제는 기존에 로컬에서 작성했던 `w1m1`, `w1m2`, `w2m5` 노트북들을 Docker 이미지로 패키징하고, EC2 인스턴스 위에서 컨테이너로 실행해 접근 가능하게 만드는 실습 과제이다.

## 이번 프로젝트를 하며 배운 점들

**1. Dockerfile을 통한 Jupyter 환경 이미지화**
`jupyter/base-notebook` 베이스 이미지에 `requirements.txt`를 복사해 필요한 라이브러리를 설치하고, `notebooks/` 폴더를 이미지 안으로 복사하는 방식으로 실행 환경 자체를 하나의 이미지로 고정하는 방법을 배웠다. 로컬 환경 설정 없이 이미지만 실행하면 동일한 노트북 환경이 재현된다는 점이 인상적이었다.

**2. requirements.txt로 의존성 버전 명시**
노트북에서 실제로 사용하는 `pandas`, `matplotlib`, `numpy`, `wordcloud`, `ipython` 등을 확인해 버전을 고정함으로써, 이미지를 다시 빌드해도 동일한 라이브러리 버전이 설치되도록 재현성을 확보하는 습관을 익혔다.

**3. user-data.sh로 EC2 초기 설정 자동화**
EC2 인스턴스가 처음 부팅될 때 Docker와 AWS CLI를 자동으로 설치하도록 `user-data.sh` 스크립트를 작성했다. 인스턴스를 새로 띄울 때마다 수동으로 Docker를 설치할 필요 없이, 부팅 스크립트만으로 컨테이너 실행 환경이 갖춰진다는 점에서 인프라 자동화의 감을 잡을 수 있었다.
