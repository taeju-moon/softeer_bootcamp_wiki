# NYC TLC 택시 데이터 분석 리포트

## 1. 클러스터 구성

| 구성 | 사양 |
|---|---|
| Master | 1대 (spark-master, `spark://spark-master:7077`, UI `:8080`) |
| Worker | 2대 (spark-worker-1/2), 각 2 core / 4g memory, UI `:8081`, `:8082` |
| Jupyter | `jupyter/pyspark-notebook:spark-3.5.0`, UI `:8888` |
| Base 이미지 | `eclipse-temurin:11-jdk-jammy` + Spark 3.5.0 (Hadoop3) + Python 3.11 |
| 볼륨 | `./data → /opt/spark-data`(입력), `./output → /opt/spark-output`(출력), `./jobs → /opt/spark-apps`, 로그 디렉터리 |
| 헬스체크 | master `curl :8080` 성공 후 worker/jupyter가 `service_healthy` 조건으로 기동 |

## 2. 데이터 파이프라인

```
Parquet 적재 (yellow_tripdata_2026-*)
  → 결측치 확인
  → trip_duration_min 계산 (dropoff - pickup)
  → 이상치 필터링 (거리 0~100mi, 소요시간 0~240min, 승객수>0, 요금>0)
  → null 처리 (핵심 컬럼 dropna, passenger_count는 1로 채움)
  → cache
  → 평균 지표 산출 + pickup_hour 추출
  → 시간대별 집계/시각화 → 피크 시간 판정
  → Open-Meteo 날씨 API 조인 (시간 단위)
  → 상관관계 분석 + 산점도
  → 결과 저장 (/opt/spark-output)
```

## 4. 해석

온도와의 상관관계(0.287)가 가장 뚜렷하며 약한 양의 상관을 보인다. 날씨가 따뜻할수록 이동 수요(도보/자전거 대체 등)가 늘 수 있는데도 택시 승차량이 같이 늘어난 것은, 기온이 낮과 활동 시간대(주간/저녁 러시아워)와도 겹치기 때문으로 보인다. 강수량·강설량은 약한 음의 상관(-0.05~-0.13)으로, 궂은 날씨에는 외출 자체가 줄어드는 효과가 대체 수단 감소 효과보다 커 보인다. 풍속은 상관이 거의 없다(0.10).

## 5. 트러블슈팅

| 문제 | 원인 | 해결 |
|---|---|---|
| Python 버전 불일치 (driver 3.11 vs executor 3.10) | Jupyter 이미지는 Python 3.11 사용, Ubuntu 22.04 기본 apt python3는 3.10 | Dockerfile에서 deadsnakes PPA로 python3.11 설치 후 python/python3 심볼릭 링크 통일 |
| Spark write 시 `NoSuchElementException: None.get` (BasicWriteJobStatsTracker) | worker 컨테이너에 `/opt/spark-output`이 마운트되어 있지 않아 driver와 output 경로가 공유되지 않음 → write 커밋 후 stats 수집 단계에서 파일을 찾지 못함 | docker-compose.yml에서 master/worker1/worker2/jupyter 전 서비스에 `./output:/opt/spark-output` 볼륨을 동일하게 추가 |
| worker가 spark-master보다 먼저 뜨며 접속 실패 | `depends_on`이 컨테이너 기동 순서만 보장, master 프로세스 준비까지는 대기 안 함 | master에 healthcheck 추가, worker/jupyter는 `condition: service_healthy`로 대기 |

## 6. 재현 방법

```bash
# 1. 클러스터 빌드 및 기동
docker-compose up -d --build

# 2. 상태 확인
docker-compose ps
docker logs spark-master --tail 50

# 3. Jupyter 접속 (브라우저)
http://localhost:8888   # notebooks/helloworld.ipynb 실행

# 4. (또는) CLI에서 노트북 실행
docker exec jupyter bash -lc \
  "source /usr/local/bin/before-notebook.d/spark-config.sh; \
   jupyter nbconvert --to notebook --execute --inplace /home/jovyan/work/helloworld.ipynb"

# 5. 종료
docker-compose down
```
