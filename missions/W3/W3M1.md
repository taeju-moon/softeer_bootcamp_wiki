# Single-Node Hadoop Cluster on Docker

컨테이너 하나에 NameNode, DataNode, ResourceManager, NodeManager를 다 띄우는 의사분산 모드 구성이다.

## 1. 사전 준비물

Docker Desktop 설치돼 있어야 한다.

```bash
docker --version
```

## 2. 파일 구성

Dockerfile, core-site.xml, hdfs-site.xml, mapred-site.xml, yarn-site.xml, entrypoint.sh, README.md 전부 같은 디렉토리에 둔다.

```
missions/W3/
├── Dockerfile
├── core-site.xml
├── hdfs-site.xml
├── mapred-site.xml
├── yarn-site.xml
├── entrypoint.sh
└── README.md
```

## 3. 이미지 빌드

```bash
cd missions/W3
docker build -t single-node-hadoop .
```

Ubuntu 20.04 위에 openjdk-8, openssh, Hadoop 3.3.6 깔고, 설정 파일 4개 복사하고, entrypoint.sh 심어놓는 과정이다. 처음 빌드하면 Hadoop tar 파일 받느라 몇 분 걸린다.

## 4. 볼륨 생성

컨테이너 지웠다 다시 만들어도 HDFS 데이터 안 날아가게 볼륨부터 만든다.

```bash
docker volume create hadoop_namenode
docker volume create hadoop_datanode
```

## 5. 컨테이너 실행

```bash
docker run -d \
  --name hadoop-cluster \
  -p 9870:9870 \
  -p 8088:8088 \
  -p 9864:9864 \
  -p 8020:8020 \
  -v hadoop_namenode:/hadoop/dfs/name \
  -v hadoop_datanode:/hadoop/dfs/data \
  single-node-hadoop
```

포트 용도:

| 포트 | 용도 |
|---|---|
| 9870 | HDFS NameNode 웹 UI |
| 8088 | YARN ResourceManager 웹 UI |
| 9864 | DataNode 웹 UI |
| 8020 | HDFS RPC |

## 6. 실행 확인

```bash
docker logs hadoop-cluster
docker exec hadoop-cluster jps
```

jps 찍었을 때 NameNode, DataNode, ResourceManager, NodeManager 다 보여야 정상이다. SecondaryNameNode까지 보이면 더 정상.

TTY 없는 환경(스크립트, CI 등)에서는 -it 옵션 빼고 실행한다.

## 7. 웹 UI 접속

브라우저로 열어본다.

- http://localhost:9870 (HDFS)
- http://localhost:8088 (YARN)

## 8. HDFS 조작

디렉토리 생성:

```bash
docker exec hadoop-cluster hdfs dfs -mkdir -p /user/hadoop/input
docker exec hadoop-cluster hdfs dfs -ls /user/hadoop
```

파일 만들고 업로드:

```bash
docker exec hadoop-cluster bash -c 'echo "Hello, Hadoop!" > /root/sample.txt'
docker exec hadoop-cluster hdfs dfs -put /root/sample.txt /user/hadoop/input/sample.txt
```

내용 확인:

```bash
docker exec hadoop-cluster hdfs dfs -cat /user/hadoop/input/sample.txt
```

다시 로컬로 받아서 원본이랑 비교:

```bash
docker exec hadoop-cluster hdfs dfs -get /user/hadoop/input/sample.txt /root/downloaded_sample.txt
docker exec hadoop-cluster diff /root/sample.txt /root/downloaded_sample.txt
```

diff 결과 아무것도 안 뜨면 두 파일 완전히 동일하다는 뜻이다.

## 9. 영속성 테스트

```bash
docker restart hadoop-cluster
docker exec hadoop-cluster hdfs dfs -cat /user/hadoop/input/sample.txt
```

entrypoint.sh가 /hadoop/dfs/name 비어있을 때만 포맷하게 짜여 있어서, 볼륨에 데이터 남아있으면 재포맷 안 하고 그대로 이어간다.

## 10. 컨테이너 중지/삭제

```bash
docker stop hadoop-cluster
docker rm hadoop-cluster
```

볼륨(hadoop_namenode, hadoop_datanode)은 따로 안 지우면 데이터 그대로 남는다. 같은 볼륨으로 다시 run 하면 이어서 쓴다.

## 11. 설정 파일 요약

| 파일 | 주요 설정 | 설명 |
|---|---|---|
| core-site.xml | fs.defaultFS | HDFS 진입점 주소 |
| hdfs-site.xml | dfs.replication=1 | 단일 노드라 복제 1 |
| hdfs-site.xml | dfs.namenode.name.dir, dfs.datanode.data.dir | 메타데이터/블록 저장 경로 |
| mapred-site.xml | mapreduce.framework.name=yarn | MapReduce를 YARN 위에서 돌림 |
| yarn-site.xml | yarn.nodemanager.aux-services | shuffle 지원 |

## 트러블슈팅

- JAVA_HOME does not exist 에러 (Apple Silicon/arm64): Dockerfile에 JAVA_HOME이 /usr/lib/jvm/java-8-openjdk-amd64로 고정돼 있으면 arm64 맥에서 빌드할 때 실제 설치 경로(java-8-openjdk-arm64)와 안 맞아서 터진다. Dockerfile 상단에 ARG TARGETARCH를 추가하고 ENV JAVA_HOME=/usr/lib/jvm/java-8-openjdk-${TARGETARCH}로 바꾼다. 빌드 아키텍처에 맞춰 자동으로 경로가 잡힌다.
- NameNode(9870)는 열리는데 YARN 웹 UI(8088)만 안 열리는 경우: 포트 매핑은 멀쩡한데 ResourceManager가 127.0.0.1에만 바인딩돼서 생기는 문제다. yarn-site.xml의 yarn.resourcemanager.hostname=localhost 설정 탓이다. yarn-site.xml에 yarn.resourcemanager.bind-host=0.0.0.0을 추가하고 YARN을 재시작(stop-yarn.sh, start-yarn.sh)하면 0.0.0.0으로 바인딩되면서 호스트에서도 접속된다.
- docker exec -it ... jps 실행 시 "cannot attach stdin to a TTY-enabled container" 에러: 대화형 터미널(TTY)이 없는 환경에서 -it 옵션을 쓰면 발생한다. 단발성 명령이면 -it를 빼고 docker exec hadoop-cluster jps처럼 실행한다.
- jps에 DataNode가 안 뜨는 경우: 재포맷 시 clusterID 불일치로 발생한다. 볼륨을 초기화(docker volume rm hadoop_namenode hadoop_datanode)하고 다시 실행한다.
- SSH 관련 오류: service ssh start가 실패하면 docker exec -it hadoop-cluster service ssh status로 상태를 확인한다.
