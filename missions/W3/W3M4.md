# W3 - Hadoop 멀티노드 클러스터 (Docker) + 감정 분석 MapReduce

namenode 1대, worker 2대로 구성한다. replication은 2로 잡는다.

- namenode: NameNode, ResourceManager
- worker1, worker2: DataNode, NodeManager

## 폴더 구조

```
missions/W3/
├── base/Dockerfile          # Java + Hadoop + SSH 설치
├── config/                  # namenode/worker 공용 설정 (core-site, hdfs-site, mapred-site, yarn-site, workers)
├── master/                  # namenode Dockerfile + entrypoint.sh
├── worker/                  # worker Dockerfile + entrypoint.sh
├── mapreduce/                # 감정 분석 MapReduce 소스 코드
│   ├── mapper.py             # 트윗 텍스트를 긍정/부정/중립으로 분류
│   ├── reducer.py             # 분류 결과를 라벨별로 집계
│   └── input/tweeter.csv      # Sentiment140 입력 데이터
└── docker-compose.yml
```

## 빌드

```
cd missions/W3
docker build -t hadoop-base:3.3.6 -f base/Dockerfile base
docker compose build
```

base 이미지는 compose가 자동으로 안 만들어주므로 먼저 따로 빌드해야 한다.

## 실행

```
docker compose up -d
```

namenode가 뜨고 나서 worker가 붙는다. worker entrypoint에 namenode:9000 대기 로직이 있어서 순서 안 맞아도 알아서 기다린다.

## 종료

```
docker compose down       # 컨테이너만 삭제, 볼륨(데이터)은 남음
docker compose down -v    # 볼륨까지 삭제, 완전 초기화
```

## 상태 확인

```
docker exec namenode jps   # NameNode, ResourceManager 떠야 함
docker exec worker1 jps    # DataNode, NodeManager 떠야 함
docker exec worker2 jps
```

- HDFS 웹 UI: http://localhost:9870
- YARN 웹 UI: http://localhost:8088

## HDFS 사용법

```
docker exec namenode hdfs dfs -mkdir -p /user/root
docker exec namenode hdfs dfs -mkdir /input

docker cp sample.txt namenode:/tmp/sample.txt
docker exec namenode hdfs dfs -put /tmp/sample.txt /input/sample.txt

docker exec namenode hdfs dfs -ls /input
docker exec namenode hdfs dfs -cat /input/sample.txt
docker exec namenode hdfs dfs -get /input/sample.txt /tmp/downloaded.txt

docker exec namenode hdfs fsck /input/sample.txt -files -blocks -locations
```

fsck 결과에서 각 블록마다 DatanodeInfoWithStorage가 2개(worker1, worker2) 찍히면 정상이다.

## 감정 분석 MapReduce 프로그램

`mapreduce/mapper.py`, `mapreduce/reducer.py`는 Hadoop Streaming으로 돌아가는 Python MapReduce 프로그램이다. Sentiment140 데이터셋(`mapreduce/input/tweeter.csv`)의 트윗 텍스트를 긍정/부정/중립 키워드 개수로 분류(mapper)하고, 라벨별 개수를 집계(reducer)한다.

### 1. 프로그램 준비 (컴파일 대신 로컬 테스트)

Python 스크립트라 별도 컴파일은 없고, 클러스터에 올리기 전에 로컬에서 파이프로 먼저 돌려서 mapper/reducer 로직이 정상인지 확인한다.

```
docker exec worker1 bash -c \
  "head -n 1000 /mapreduce/input/tweeter.csv | python3 /mapreduce/mapper.py | sort | python3 /mapreduce/reducer.py"
```

`positive`, `negative`, `neutral` 세 줄과 각 라벨의 카운트가 출력되면 정상이다.

### 2. 입력 데이터 HDFS 업로드

`mapreduce/input`이 worker1의 `/mapreduce/input`으로 마운트되어 있으므로, 바로 HDFS로 put 할 수 있다.

```
docker exec worker1 hdfs dfs -mkdir -p /input
docker exec worker1 hdfs dfs -put /mapreduce/input/tweeter.csv /input/tweeter.csv
docker exec worker1 hdfs dfs -ls /input
```

### 3. MapReduce 잡 실행

Hadoop Streaming jar로 mapper.py/reducer.py를 각각 map/reduce 태스크로 실행한다. `-file`로 지정한 스크립트는 Hadoop이 자동으로 모든 노드에 배포해준다.

```
docker exec worker1 hadoop jar /opt/hadoop/share/hadoop/tools/lib/hadoop-streaming-3.3.6.jar \
  -input /input/tweeter.csv \
  -output /output \
  -mapper "python3 mapper.py" \
  -reducer "python3 reducer.py" \
  -file /mapreduce/mapper.py \
  -file /mapreduce/reducer.py
```

output 디렉토리가 이미 있으면 에러 나므로, 재실행할 땐 `hdfs dfs -rm -r /output` 먼저 한다.

진행 상황은 YARN 웹 UI(http://localhost:8088)에서 확인할 수 있다.

### 4. 결과 데이터 가져오기

```
docker exec namenode hdfs dfs -cat /output/part-00000
docker exec namenode hdfs dfs -get /output/part-00000 /tmp/sentiment-result.txt
docker cp namenode:/tmp/sentiment-result.txt ./sentiment-result.txt
```

`negative`, `neutral`, `positive` 라벨별 트윗 개수가 출력/저장되면 완료.

## 트러블슈팅

**`docker run IMAGE bash -c "..."`로 파일 확인하려다가 컨테이너가 안 꺼짐**
namenode/worker 이미지는 ENTRYPOINT가 `/entrypoint.sh`로 고정돼 있음. 뒤에 붙인 bash 명령이 검증용이 아니라 entrypoint.sh 인자로 들어가버려서 실제 데몬이 떠버림. 파일만 확인하고 싶으면 `--entrypoint bash`로 덮어써야 한다.

**컨테이너 재시작하면 데이터 날아가는 문제**
`hdfs namenode -format`을 매번 하면 메타데이터가 초기화됨. namenode entrypoint.sh에서 `/hadoop/dfs/name` 디렉토리가 비어있을 때만 format 하도록 체크함.

**YARN에서 멀쩡한 task가 죽는 경우**
Docker 컨테이너 안에서는 가상 메모리 사용량이 실제보다 부풀려져 보여서 YARN이 오판하고 task를 kill할 수 있음. yarn-site.xml에서 `pmem-check-enabled`, `vmem-check-enabled` 둘 다 false로 꺼둠.

**worker가 컨테이너는 떴는데 datanode가 안 뜸**
namenode의 NameNode가 완전히 준비되기 전에 worker가 datanode를 켜려고 하면 실패함. worker entrypoint.sh에 `/dev/tcp/namenode/9000` 연결될 때까지 기다리는 루프 넣음.
